## How does Smite interact with death-saving replacement effects? [#death-replacement-effects]

Death-saving effects such as <Card name="Guardian Angel" />, <Card name="Zhonya's Hourglass" />, and <Card name="Tactical Retreat" /> can prevent the banish, because the unit's controller decides which replacement effect applies first and can apply their protective effect before Smite's.

Smite's "if it would die this turn, banish it instead" is a replacement effect.<Rule number="369.1" />
So are each of the three protective effects, which all replace the event of a unit dying.
[When multiple replacement effects apply, the affected unit's controller chooses their order](/general-rules/replacement-effects#multiple-replacements).

If the unit's controller applies their protective effect first, the death event is fully replaced — the unit is healed, exhausted, and recalled, or an equipment dies in its place.<Rule number="370.1.a.1" />
A replacement effect can only be applied once to an event, and to any events that replace it;<Rule number="370.2" /> once the death event has been consumed, Smite's replacement effect has no qualifying event left to intercept, and the unit is not banished.

Conversely, if Smite's effect is applied first, the unit is banished.
The protective effects each replace a "die" event — since banishment is not a subset of killing,<Rule number="427.2.a" /> none of them can chain onto the resulting "banish" event.
